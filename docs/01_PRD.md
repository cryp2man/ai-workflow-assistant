# 1. Project Overview

AI Workflow Assistant - это демонстрационная платформа автоматизации бизнес-процессов, предназначенная для обработки входящих заявок с использованием Telegram, AI и REST API.

Проект разрабатывается как портфолио для демонстрации навыков проектирования архитектуры, автоматизации процессов, интеграции сервисов и использования LLM в реальных бизнес-сценариях.

---

# 2. Project Goals

Основные цели проекта:

- автоматизировать обработку входящих заявок;
- сократить объем ручной работы менеджеров;
- использовать LLM для анализа обращений;
- сохранить структурированные данные в базе;
- предоставить менеджеру готовое краткое резюме заявки.

---

# 3. Target Audience

Проект ориентирован на небольшие и средние компании, которые работают с входящими обращениями клиентов.

Примеры:

- агентства недвижимости;
- отделы продаж;
- консалтинговые компании;
- сервисные компании;
- стартапы.

---

# 4. Business Problem

Во многих компаниях обработка заявок выглядит следующим образом:

1. Клиент пишет менеджеру.
2. Менеджер вручную задает одинаковые вопросы.
3. Данные копируются в CRM.
4. Информация анализируется вручную.
5. Руководитель не получает единой картины.

Недостатки такого процесса:

- потеря времени;
- человеческие ошибки;
- отсутствие стандартизации;
- медленная обработка лидов.

---

# 5. Proposed Solution

Система автоматизирует полный цикл обработки обращения.

Общий процесс:

1. Пользователь начинает диалог с Telegram-ботом.
2. Бот собирает необходимую информацию.
3. Backend валидирует данные.
4. Данные сохраняются в PostgreSQL.
5. LLM анализирует заявку.
6. AI формирует:
   - краткое описание;
   - категорию;
   - приоритет;
   - рекомендацию менеджеру.
7. Менеджер получает уведомление.
8. Заявка появляется в Dashboard.

---

# 6. Functional Requirements

## Telegram Bot

Пользователь может:

- начать новую заявку;
- заполнить анкету;
- подтвердить отправку данных;
- получить сообщение об успешной регистрации заявки.

---

## Backend API

Backend должен:

- принимать данные;
- валидировать информацию;
- сохранять данные;
- вызывать AI;
- возвращать результаты анализа.

---

## AI Module

LLM должна:

- анализировать текст заявки;
- создавать Summary;
- определять категорию;
- определять приоритет;
- формировать рекомендацию.

---

## Dashboard

Менеджер должен иметь возможность:

- просматривать заявки;
- искать заявки;
- фильтровать заявки;
- изменять статус заявки;
- просматривать AI-анализ.

---

# 7. Non-functional Requirements

Система должна:

- иметь модульную архитектуру;
- быть легко расширяемой;
- использовать Docker;
- иметь централизованное логирование;
- использовать конфигурацию через .env;
- корректно обрабатывать ошибки.

---

# 9. MVP Scope

В первую версию входят:

- Telegram Bot;
- FastAPI Backend;
- PostgreSQL;
- AI-анализ заявки;
- Dashboard;
- REST API;
- Docker Compose.

---

# 10. Out of Scope

В первую версию не входят:

- авторизация пользователей;
- роли и права доступа;
- работа с несколькими компаниями;
- платежные системы;
- интеграции с внешними CRM;
- мобильное приложение.

---

# 11. Success Criteria

Проект считается успешно завершенным, если:

- пользователь может оставить заявку через Telegram;
- заявка сохраняется в базе данных;
- AI успешно анализирует заявку;
- менеджер получает уведомление;
- Dashboard отображает созданную заявку;
- проект запускается через Docker Compose.

---

# 12. Future Improvements
