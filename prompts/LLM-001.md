LLM-001 - создать базовый LLM Provider

Контекст проекта

ExecutionEngine уже существует.

В ближайших задачах движок начнёт использовать LLM.

Перед этим необходимо создать абстракцию, чтобы Engine никогда не зависел напрямую от OpenAI, Gemini или Claude.

========================================

Работаем строго по принципу Surgical.

Разрешается создать только

src/llm/

и файлы внутри этой директории.

Не изменять

Engine

Services

Repositories

Models

Schemas

API

Alembic

Tests

========================================

Создать структуру

src/llm/

с файлами

__init__.py

base.py

========================================

В base.py создать

абстрактный класс

BaseLLMProvider

используя

ABC

и

abstractmethod

========================================

Определить один метод

async def generate(
    self,
    prompt: str,
) -> str

Метод должен быть абстрактным.

Никакой реализации.

========================================

В __init__.py экспортировать

BaseLLMProvider

========================================

После выполнения проверить

import src.main

uv run ruff check src/llm

========================================

В отчёте указать

1. Какие файлы созданы.

2. Почему используется ABC.

3. Почему Engine не должен знать про OpenAI.

4. Почему метод generate() пока пустой.

5. Проходит ли импорт.

6. Результат Ruff.

Никаких дополнительных улучшений не выполнять.
