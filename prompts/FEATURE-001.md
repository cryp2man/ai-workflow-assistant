==================================================
FEATURE-001
Интеграция настоящего Ollama
==================================================

Цель

Сейчас OllamaProvider умеет только вызывать AsyncOpenAI.

Нужно сделать полноценную рабочую интеграцию через настройки проекта.

==================================================
Разрешается изменить
==================================================

src/core/config.py

src/llm/openai_provider.py

src/engine/execution_engine.py
(только если потребуется для совместимости)

.env.example

==================================================
Что необходимо сделать
==================================================

1. Добавить в Settings параметры:

OLLAMA_BASE_URL

OLLAMA_MODEL

OLLAMA_TIMEOUT

используя pydantic-settings в стиле существующего проекта.

==================================================

2. Обновить .env.example

Добавить

OLLAMA_BASE_URL=http://localhost:11434/v1

OLLAMA_MODEL=qwen3:8b

OLLAMA_TIMEOUT=120

==================================================

3. Переделать OllamaProvider.

Он должен принимать Settings через конструктор.

Не хранить base_url и model внутри кода.

Использовать значения из Settings.

==================================================

4. AsyncOpenAI должен создаваться так:

base_url ← Settings

api_key="ollama"

timeout ← Settings

==================================================

5. generate()

использует

settings.OLLAMA_MODEL

вместо захардкоженного значения.

==================================================

6. Никакой новой логики.

Никаких retry.

Никаких fallback.

Никаких stream.

==================================================

7. ExecutionEngine не должен знать:

ни URL,

ни model,

ни timeout.

Только BaseLLMProvider.
