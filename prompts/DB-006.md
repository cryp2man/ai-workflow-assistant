1. Создай Alembic migration для новой модели WorkflowStep.
2. Используй только autogenerate.
3. Не изменяй вручную существующие миграции.

5. После генерации примени миграцию к основной базе.
6. Выполни:
   - alembic current
   - alembic check

   - Primary Key
   - Foreign Key на workflows.id

    - revision;
    - какие таблицы созданы;
    - какие индексы созданы;
    - какие foreign keys появились;
    - результат alembic current;

    - какие файлы были изменены.
