ENGINE-003 - завершение WorkflowRun

Контекст проекта

ExecutionEngine уже умеет

- получать Workflow
- создавать WorkflowRun
- сохранять его со статусом "running"

Необходимо реализовать завершение выполнения.

========================================

Работаем строго по принципу Surgical.

Разрешается изменить только

src/engine/execution_engine.py

========================================

Не изменять

Repository

Service

Models

Schemas

Alembic

API

========================================

Алгоритм execute_workflow()

после создания WorkflowRun

должен продолжить выполнение.

========================================

Пока никакого AI.

Никаких HTTP.

Никаких внешних вызовов.

========================================

После создания WorkflowRun

необходимо

изменить поля объекта

status = "completed"

finished_at = datetime.utcnow()

result = {
    "message": "Workflow executed successfully"
}

========================================

После изменения объекта

использовать

workflow_run_service.repository.session.commit()

для сохранения изменений.

Никаких новых методов Service и Repository не создавать.

========================================

После commit()

вернуть WorkflowRun.

========================================

try/except

пока НЕ использовать.

========================================

raise ValueError("Workflow not found")

import src.main

uv run ruff check src/engine
