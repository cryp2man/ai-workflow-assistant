==================================================
FEATURE-005

Workflow Step API (полный CRUD)

==================================================

Цель

Сделать возможным полностью управлять WorkflowStep через REST API.

После выполнения задачи пользователь должен иметь возможность:

создать шаг

получить список шагов Workflow

получить один шаг

обновить шаг

удалить шаг

без прямой работы с БД.

==================================================

Использовать существующие:

WorkflowStepRepository

WorkflowStepService

никакой новой бизнес-логики.

==================================================

/api/v1/workflows/{workflow_id}/steps

--------------------------------------------------

PATCH

/api/v1/workflow-steps/{step_id}

--------------------------------------------------

DELETE

/api/v1/workflow-steps/{step_id}

==================================================

Создать необходимые Pydantic Schema

WorkflowStepCreate

WorkflowStepUpdate

WorkflowStepResponse

в стиле остальных схем проекта.

==================================================

POST принимает

{

"title": "...",

"prompt": "...",

"step_order": 1

}

workflow_id брать только из Path.

==================================================

PATCH

==================================================
