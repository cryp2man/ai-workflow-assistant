# TASK-005 - API Structure

## Контекст

Проект AI Workflow Assistant.

Уже реализовано:

- структура проекта;
- FastAPI;
- конфигурация приложения;
- logging;
- lifespan.

Структура каталогов уже создана вручную:

src/
└── api/
    ├── __init__.py
    ├── router.py
    └── v1/
        ├── __init__.py
        ├── router.py
        └── endpoints/
            └── __init__.py

Не изменять существующую структуру.

---

## Цель

Настроить маршрутизацию API без создания конечных точек.

---

## Необходимо выполнить

1. Настроить главный APIRouter в `src/api/router.py`.

2. Создать роутер версии API в `src/api/v1/router.py`.

3. Подключить роутер v1 к главному роутеру.

4. Подключить главный роутер в `src/main.py`.

5. Использовать префикс:

/api/v1

6. Пока НЕ создавать никаких endpoints.

7. Swagger должен успешно открываться.

---

## Ограничения

Не создавать:

- /health
- /ping
- тестовые маршруты
- middleware
- зависимости
- сервисы
- модели

---

## Acceptance Criteria

После запуска:

uv run uvicorn src.main:app --reload

должно выполняться:

- приложение запускается без ошибок;
- Swagger доступен;
- пользовательских endpoints пока нет;
- структура проекта не изменилась.

---
