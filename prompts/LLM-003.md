LLM-003 - реализовать OllamaProvider

Контекст проекта

В проекте уже существуют:

- BaseLLMProvider
- OpenAIProvider (заглушка)
- PromptBuilder
- ExecutionEngine

Проект должен использовать бесплатные инструменты.

Вместо OpenAI API необходимо реализовать провайдер для Ollama.

========================================

Работаем строго по принципу Surgical.

Разрешается изменить только

src/llm/openai_provider.py

src/llm/__init__.py

========================================

Не изменять

Engine

PromptBuilder

Repositories

Services

Models

Schemas

API

Alembic

Tests

========================================

Переименовать класс

OpenAIProvider

в

OllamaProvider

========================================

Использовать официальный пакет

openai

(версии 1.x)

========================================

В конструкторе создать

AsyncOpenAI

с параметрами

base_url="http://localhost:11434/v1"

api_key="ollama"

и сохранить клиент в

self.client

Параметр api_key конструктора оставить для совместимости, но не использовать.

========================================

Реализовать

async def generate(
    self,
    prompt: str,
) -> str

Использовать

await self.client.chat.completions.create()

========================================

Параметры

model="qwen3:8b"

messages=[
    {
        "role": "user",
        "content": prompt,
    }
]

========================================

Вернуть

response.choices[0].message.content

Если content == None

вернуть пустую строку.

========================================

Никаких retry.

Никаких timeout.

Никаких обработчиков ошибок.

Никаких логгеров.

========================================

В __init__.py экспортировать

OllamaProvider

вместо

OpenAIProvider

========================================

После выполнения проверить

import src.main

uv run ruff check src/llm

========================================

В отчёте указать

1. Какие изменения внесены.

2. Почему выбран Ollama.

3. Почему используется AsyncOpenAI.

4. Почему выбран model="qwen3:8b".

5. Проходит ли импорт.

6. Результат Ruff.

Если пакет openai отсутствует или Ollama не установлен — ничего автоматически не исправлять, а указать это в отчёте.

Никаких дополнительных улучшений не выполнять.
