DB-005 - Alembic migration для WorkflowRun

Контекст проекта

FastAPI
SQLAlchemy 2.x
Alembic
PostgreSQL
Async

В проекте уже существует модель:

src/db/models/workflow_run.py

которая пока отсутствует в базе данных.

Необходимо создать миграцию Alembic.

==========================
Что нужно сделать
==========================

1. Создать новую Alembic migration через autogenerate. Название: feat_create_workflow_runs_table

2. Миграция должна создать таблицу workflow_runs со всеми полями модели.

3. Должны быть созданы:

Primary Key

Foreign Key:

workflow_id → workflows.id

Индекс на workflow_id

4. Никаких ручных SQL. Использовать стандартный Alembic autogenerate.

5. После генерации проверить migration. Если Alembic сгенерировал что-либо лишнее -
не исправлять ничего автоматически. Только сообщить.

==========================
Проверки
==========================

После генерации выполнить:

uv run alembic check

затем

uv run alembic upgrade head

После этого проверить:

uv run alembic current

и сообщить:

- название migration
- revision id
- head
- какие таблицы появились
- какие индексы появились
