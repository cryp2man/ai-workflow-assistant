SER-003 - WorkflowService CRUD.

ВАЖНО.

Работаем строго по принципу Surgical.

========================================

Разрешается изменить только

- src/services/workflow_service.py

========================================

Запрещается менять

- Repository
- Models
- API
- Dependencies
- Alembic
- Database
- Schemas
- Tests

========================================

Цель

Реализовать сервисный слой для WorkflowRepository по аналогии с UserService.

========================================

Необходимо реализовать методы

- create_workflow(workflow: Workflow) -> Workflow
- get_workflow(workflow_id: int) -> Workflow | None
- list_workflows() -> list[Workflow]
- list_user_workflows(user_id: int) -> list[Workflow]
- delete_workflow(workflow: Workflow) -> None

========================================

Требования

Мутирующие методы

- create_workflow()
- delete_workflow()

должны

- вызывать repository
- выполнять await session.commit()
- при IntegrityError выполнять rollback()
- затем повторно выбрасывать исключение

Read-only методы

- get_workflow()
- list_workflows()
- list_user_workflows()

должны только делегировать вызов репозиторию.

========================================

Использовать

from sqlalchemy.exc import IntegrityError

Никаких собственных исключений.

Никакой новой бизнес-логики.

========================================

После выполнения проверить

- import src.main
- uv run ruff check src/services/workflow_service.py

========================================

В отчёте указать

1. Какие методы реализованы.

2. Где используется commit().

3. Где используется rollback().

4. Почему commit находится именно в Service.

5. Какие исключения обрабатываются.

6. Проходит ли импорт проекта.

7. Результат Ruff.

Никаких дополнительных улучшений не делать.
