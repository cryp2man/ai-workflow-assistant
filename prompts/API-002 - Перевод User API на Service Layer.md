
ВАЖНО:

Работаем строго по принципу Surgical Changes.

Никаких улучшений, рефакторингов, форматирований, оптимизаций и изменений вне задания.

Если увидишь возможность что-то улучшить - НЕ ДЕЛАЙ этого.

Если для выполнения задачи потребуется решение, которого нет в задании - сначала остановись и спроси меня.

===========================================
ЗАДАНИЕ
===========================================

Необходимо перевести API пользователей на Service Layer.

Использовать уже существующие:

- UserService
- get_user_service()

Нельзя использовать UserRepository напрямую внутри endpoint.

===========================================
Изменить только:

src/api/v1/users.py

===========================================

Что требуется:

1.

Во всех endpoint заменить зависимость

Depends(get_db)

или

Depends(get_user_repository)

на

Depends(get_user_service)

если они используются.

2.

В endpoint использовать методы сервиса:

создание:

service.create_user(...)

получение:

service.get_user(...)

по telegram:

service.get_user_by_telegram(...)

список:

service.list_users()

удаление:

service.delete_user(...)

3.

Никакой SQLAlchemy внутри endpoint.

Никакого session.

Никакого commit.

Никакого rollback.

Endpoint вызывает только сервис.

4.

HTTPException оставить в endpoint.

Если сервис вернул None - endpoint возвращает:

404 Not Found

5.

Schemas менять нельзя.

Repository менять нельзя.

Service менять нельзя.

Database менять нельзя.

Alembic менять нельзя.

===========================================
После выполнения:

1.

Покажи полный отчёт:

- какие файлы изменены;

- какие endpoint переведены;

- какие зависимости теперь используются;

- изменилось ли поведение API.

2.

Проверь:

- import src.main

- Swagger

- Ruff только изменённого файла

3.

Не делать commit.

Не менять CHANGELOG.

Не менять TASKS.
