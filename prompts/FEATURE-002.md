==================================================
FEATURE-002
API запуска Workflow
==================================================

Цель

Добавить HTTP API для запуска Workflow через существующий ExecutionEngine.

После выполнения должно быть возможно:

POST /api/v1/workflows/{workflow_id}/execute

и получить результат выполнения.

==================================================
Разрешается изменить
==================================================

src/api/v1/

src/api/v1/__init__.py
(если потребуется)

src/api/v1/router.py
(если потребуется)

src/dependencies.py

==================================================
Что необходимо сделать
==================================================

1. Создать новый endpoint

POST

/api/v1/workflows/{workflow_id}/execute

==================================================

2. Endpoint получает

workflow_id

из Path.

Body отсутствует.

==================================================

3. ExecutionEngine получать через Depends.

Не создавать его вручную внутри endpoint.

==================================================

4. Вызвать

await execution_engine.execute_workflow(workflow_id)

==================================================

5. Если Workflow отсутствует

(ValueError)

вернуть

404

с сообщением

"Workflow not found"

==================================================

6. Если выполнение успешно

вернуть HTTP 200

JSON

{
    "run_id": ...,
    "status": "...",
    "result": ...
}

где значения берутся из WorkflowRun,
возвращённого ExecutionEngine.

==================================================

7. Никакой дополнительной бизнес-логики.

Никаких try/except кроме обработки ValueError.

Никаких новых сервисов.

Никаких новых Repository.
