TEST-005 - Test Database Setup.

ВАЖНО.

Работаем строго по принципу Surgical.

========================================

НЕ менять

- src/
- API
- Repository
- Service
- Models
- Alembic
- Docker compose основной базы

========================================

Разрешается изменить только

docker/

compose.yaml (если необходимо исключительно для test_db)

tests/

========================================

Цель

Подготовить отдельную тестовую PostgreSQL базу для pytest.

Production база использоваться НЕ должна.

========================================

Необходимо

1. Создать отдельную базу

test_db

2. Создать пользователя

test_user

с отдельным паролем.

3. Обновить

tests/conftest.py

так, чтобы TEST_DATABASE_URL использовал test_user и test_db.

4. Если используется Docker, использовать существующий PostgreSQL-контейнер. Не создавать второй контейнер PostgreSQL без необходимости. Если test_db можно создать внутри существующего контейнера - использовать этот вариант.

5. Не выполнять create_all(). Не создавать таблицы автоматически. База должна быть пустой.

========================================

После настройки

проверить

что pytest может успешно подключиться к test_db.

Если тесты падают уже по причине отсутствия таблиц - это считается корректным результатом.

========================================

В отчёте указать

1. Что создано.

2. Какая строка подключения используется.

3. Удалось ли подключиться.

4. На каком этапе теперь останавливаются тесты.

Никаких дополнительных улучшений не делать.
