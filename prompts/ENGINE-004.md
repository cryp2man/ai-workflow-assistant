ENGINE-004 - интеграция PromptBuilder в ExecutionEngine

Контекст проекта

В проекте уже существуют:

- ExecutionEngine
- PromptBuilder
- BaseLLMProvider
- OpenAIProvider

Необходимо начать использовать PromptBuilder внутри движка.

========================================

Работаем строго по принципу Surgical.

Разрешается изменить только

src/engine/execution_engine.py

========================================

Не изменять

LLM

PromptBuilder

Repositories

Services

Models

Schemas

API

Alembic

Tests

========================================

В ExecutionEngine

добавить зависимость

PromptBuilder

через конструктор.

Сохранить её как

self.prompt_builder

========================================

В execute_workflow()

после получения Workflow

создать простой prompt.

Использовать

workflow.title

если поле существует.

Если поля нет - использовать строку

"Execute workflow"

========================================

Перед созданием WorkflowRun

вызвать

self.prompt_builder.build_prompt(...)

и сохранить результат

в локальную переменную

prompt

Пока этот prompt нигде не использовать.

Он понадобится в следующих задачах.

========================================

Никаких вызовов LLM.

Никаких изменений WorkflowRun.

Никакой новой логики.

========================================

После выполнения проверить

import src.main

uv run ruff check src/engine

========================================

В отчёте указать

1. Какие изменения внесены.

2. Почему PromptBuilder внедряется через конструктор.

3. Почему prompt пока только формируется.

4. Почему Engine не должен собирать строки самостоятельно.

5. Проходит ли импорт.

6. Результат Ruff.

Никаких дополнительных улучшений не выполнять.
