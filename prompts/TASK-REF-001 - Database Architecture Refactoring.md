
## Цель

Выполнить небольшой архитектурный рефакторинг Database Layer без изменения поведения приложения.

Важно:

- НЕ менять бизнес-логику.
- НЕ менять API.
- НЕ создавать новые модели.
- НЕ менять миграции Alembic.
- НЕ изменять README.
- НЕ добавлять новые библиотеки.

---

# Часть 1

Подготовить models/__init__.py

Сейчас модели импортируются напрямую в alembic/env.py.

Необходимо централизовать экспорт моделей.

Файл:

src/db/models/__init__.py

должен экспортировать все существующие модели.

Например:

User

Workflow

(если существуют другие модели - тоже экспортировать.)

---

После этого изменить alembic/env.py.

Вместо отдельных импортов моделей использовать единый импорт из:

src.db.models

Base.metadata должна продолжать корректно содержать все таблицы.

Autogenerate Alembic должен продолжать работать.

---

# Часть 2

Упростить env.py.

Сейчас DATABASE_URL получается двумя способами:

через dotenv

и через Settings.

Необходимо оставить только один источник правды.

Использовать только:

settings.DATABASE_URL

Удалить:

load_dotenv()

os.getenv()

и весь связанный код.

Конфигурация должна работать исключительно через Settings.

---

# Часть 3

Проверить session.py

Ничего кардинально не менять.

Только проверить:

- create_async_engine
- async_sessionmaker
- AsyncSession

Если всё корректно - не изменять.

---

# Часть 4

Проверить Base.

Если Base реализован через DeclarativeBase - ничего не менять.

---

# Часть 5

Ничего не менять в моделях User и Workflow.

UUID будем внедрять отдельной задачей.
