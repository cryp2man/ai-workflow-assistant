# Project Rules

## Development

- Документация первична.
- Код пишется только после утверждения документации.
- Одна задача = один коммит.
- Все изменения проходят локальное тестирование.

## AI Usage

- AI используется как инструмент разработки.
- Все архитектурные решения принимаются человеком.
- Любой сгенерированный код проходит проверку.
- Код не копируется без понимания.

## Git

- Feature Branch Workflow.
- Осмысленные commit messages.
- Pull Request перед Merge.

## Documentation

- README обновляется перед релизом.
- CHANGELOG обновляется после каждого этапа.
- Документация синхронизируется с кодом.
